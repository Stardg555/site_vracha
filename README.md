# site_vracha
Тестовое задание по доработке решения "Сайт врача"

Контекст

У нас есть продукт __«Персональный сайт врача»__, разработанный на базе 1С\-Битрикс\.

В продукте уже присутствует базовый модуль записи на приём к врачу, однако сейчас он реализован в упрощённом виде — фактически на уровне вёрстки и без административного интерфейса для управления расписанием\.

Посмотреть решение можно здесь:  
[https://marketplace\.1c\-bitrix\.ru/solutions/vit\.doctor/\#tab\-comments\-link](https://marketplace.1c-bitrix.ru/solutions/vit.doctor/#tab-comments-link)

Решение можно установить бесплатно и использовать для выполнения тестового задания\.

Задача

Необходимо реализовать __управление расписанием врача__\.

Нужно создать интерфейс в Bitrix, в котором можно настроить __рабочую неделю врача__:

— задать рабочие дни недели

— указать время начала и окончания работы

Пример:

Понедельник — 09:00–18:00  
Вторник — 09:00–18:00  
Среда — выходной

Важно

Текущую реализацию модуля можно не использовать\.

Допускается предложить и реализовать __собственную архитектуру хранения и управления расписанием__\.

Важно кратко обьяснить выбранный подход в решении\.

Ограничение по времени

Ориентир на выполнение задания — __до 3 часов__\.

Если решение не завершено полностью, можно прислать частично готовый результат\.

— короткое описание архитектуры решения
